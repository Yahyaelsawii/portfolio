01

The Problem

The existing login and registration experience wasn’t optimized for mobile, and the registration page was too long, creating friction and increasing time-to-complete.

  • Not mobile optimized: dense layout, harder typing, and unclear progression.
  • Registration was long: took around 3–4 minutes to finish.
  • Platform inconsistency: mobile and desktop didn’t use the same API, creating mismatched behaviors.

Goals

The redesign was driven by one main idea: simplicity. Reduce steps, reduce typing, reduce mistakes.

speed

Faster registration

Reduce completion time from 3–4 minutes to 1–2 minutes by shortening and restructuring the flow.

smartphone

Mobile-first clarity

Improve readability, spacing, and tap targets to make the flow comfortable on mobile.

mark_email_read

Passwordless login

Add “Login with email” where users receive a code, no password required.

lan

Unified platform behavior

Ensure mobile and desktop use the same API for consistent experiences and fewer edge-case bugs.

03

The Solution

I redesigned the experience end-to-end with a “less is more” approach, removing unnecessary steps, introducing passwordless login, and designing clear states for validation and errors.

format_list_bulleted

Shorter registration

Broke long forms into cleaner steps and reduced “decision fatigue” on mobile.

pin

Email OTP login

Users enter their email → receive a code → verify and continue (no password entry).

rule

Clear error states

Designed predictable feedback for invalid codes, resend timers, and missing captcha acceptance.

sync

Same API across platforms

The new design aligned web and mobile to use the same backend API for consistent behavior.

Design principle

“Simplicity wins.” Each screen should ask the user to make one clear decision and move forward with minimal typing.

Notes & Learnings

1

Simplicity is a feature

Cutting steps and making states obvious improves both speed and confidence, especially on mobile.

2

Consistency reduces bugs

Aligning web and mobile on the same API supports predictable UX and reduces edge-case inconsistencies.
