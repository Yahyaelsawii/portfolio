01

Funnel + Drop-offs

The checkout funnel showed the biggest drop-offs around shipping/contact details, payment selection, and the e-invite setup step. The goal was to reduce friction and improve confidence at each step.

  • Key funnel steps: Product → Amount → Recipient → Payment → Confirmation
  • Hypothesis: unclear steps + trust questions cause abandonment
02

Friction Points

After reviewing the flow and support patterns, the biggest friction areas were clarity, trust, and unnecessary steps.

account_circle

Too much input

Too many fields before users understand what happens next (code delivery, recipient flow, refund rules).

shield

Trust questions

Users need reassurance: “When do I receive the code?” “Is this legit?” “Can I gift it easily?”

mail

Post-purchase gap

Even when checkout succeeds, confusion can happen if the confirmation state and transactional emails don’t clearly explain order status, delivery, and “who receives what”.

(This ties into your transactional email work: order confirmations + e-invites.)

03

Redesigned Flow

The redesign simplifies steps, adds reassurance at key moments, and makes e-invite feel intentional, not “extra work”.

Change #1

Step clarity

Add a stepper + “What happens next” micro-copy.

Change #2

Trust blocks

Code delivery + refund policy + support visible early.

Change #3

E-invite defaults

Smart defaults + optional personalization (message, schedule, preview).

04

A/B Test Plan

Primary metrics

  • • Checkout completion rate
  • • Payment step completion
  • • E-invite completion (if selected)
  • • Time to complete purchase

Secondary metrics

  • • Support tickets “Where is my code?”
  • • Email open rate & click rate
  • • Refund/dispute rate
  • • NPS / post-purchase feedback

06

Transaction Emails + Order Confirmation UX

To reduce “Where is my code?” confusion, we treated the confirmation page and emails as part of the product, not an afterthought.

  • Order confirmation: clear status, expected delivery, and next actions.
  • E-invite email: sender/recipient clarity + what happens when opened.
  • Order updates: payment confirmed → code delivered → resend/support.
