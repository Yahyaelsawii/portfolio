The Problem

We designed and implemented an Oracle relational database to support these workflows and enable real-time operational visibility through queries and views.

05

Backend Highlights

badge Users & Privileges

We created multiple DB users with realistic permissions (SYS, DEV_USER, CUSTOMER_USER, EMPLOYEE_USER, SUPERVISOR_USER) to enforce least-privilege access.

lan Remote Connections

Verified secure remote access for different users while keeping permission boundaries intact.

code Queries & Views

Built SQL queries for operational reporting (rentals by customer/employee, vehicles by price, revenue), and created a rental summary view to support dashboard-style reporting.

sync_alt Transactions

Implemented core transactions for renting a vehicle, inserting average daily cost records, and logging maintenance events.

cloud_upload Backup Strategy

Designed a cold backup approach and automated upload of backups to S3 to support recovery and reliability.

All backend features above are documented in the project report (schema/ERD, users & privileges, remote access, queries/views, transactions, backup).
